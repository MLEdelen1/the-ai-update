WORKFLOW

Weekly Review

Run a fast weekly debrief that improves execution instead of repeating the same misses.

What this solves

This workflow captures what worked, what failed, and why, then converts lessons into concrete changes for next week.

When to use / when not to use

  • Use every Friday afternoon or Sunday planning block.
  • Do not use as a performance review for other people.

Inputs needed

  • Completed tasks and unfinished tasks
  • Calendar data and interruptions
  • Any metrics you track weekly

Copy-ready prompt block

Run a weekly performance review on my data.

Return these sections:
1) Wins
2) Misses
3) Root causes
4) What to stop, start, continue
5) Next week plan with top 3 outcomes

Data:
Completed:
[paste]
Unfinished:
[paste]
Notes/metrics:
[paste]

Execution steps

  1. Collect completed and incomplete work from your systems.
  2. Run the prompt and review root causes honestly.
  3. Pick one process change for each root cause.
  4. Add top three outcomes to next week calendar.

Expected output format

Wins
- ...

Misses
- ...

Root Causes
- ...

Stop / Start / Continue
Stop: ...
Start: ...
Continue: ...

Next Week Top 3 Outcomes
1) ...
2) ...
3) ...

Concrete example

Review shows repeated missed deadlines caused by afternoon meeting spillover. Next-week correction blocks deep work from 9:00 to 11:00 daily and moves status calls after lunch.

Common mistakes

  • Listing misses without identifying root causes
  • Setting too many goals for next week
  • Ignoring calendar reality

Quick quality check

  • Root causes are specific
  • Each cause has one correction
  • Next week has only three outcomes
  • Corrections are scheduled in calendar

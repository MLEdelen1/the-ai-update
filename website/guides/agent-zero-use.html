USE GUIDE ? AGENT ZERO

Main takeaway: run Agent Zero with Docker and give clear tasks

Docker is a container tool that runs apps in isolated packages across Windows, macOS, and Linux.

WINDOWS / MACOS / LINUX

Check running container

docker ps

Expected result: you see agent0ai/agent-zero in container list.

UI

Open local app

http://localhost:50001

Expected result: Agent Zero interface opens in browser.

FIRST TASK

Use a structured prompt

Goal: summarize these support tickets.
Context: today's queue.
Output: top 5 issues and one action each.

Expected result: cleaner output than a vague prompt.

STOP

Stop container after work

docker stop $(docker ps -q --filter ancestor=agent0ai/agent-zero)

Expected result: container stops and resources are freed.

TROUBLESHOOTING

Common fixes

If localhost:50001 does not load, rerun container with docker run -p 50001:80 agent0ai/agent-zero.

If port 50001 is busy, use -p 50002:80 and open port 50002 in browser.

If image is missing, run docker pull agent0ai/agent-zero.

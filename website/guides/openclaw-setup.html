OpenClaw setup from zero to stable daily use

Install OpenClaw, start the gateway, run your first task, then lock down a clean baseline with verification and troubleshooting commands.

STEP 1

Install prerequisites

Use Node.js LTS 20+ and Git. Keep OpenClaw in user scope first so updates are simple.

Windows PowerShell

winget install OpenJS.NodeJS.LTS
winget install Git.Git
node -v
npm -v
git --version

macOS/Linux (optional)

brew install node git
node -v && npm -v && git --version
STEP 2

Install OpenClaw and verify CLI

Global npm install gives you the openclaw command in every shell.

Windows PowerShell

npm install -g openclaw
openclaw help
openclaw gateway status

macOS/Linux (optional)

npm install -g openclaw
openclaw help
openclaw gateway status
STEP 3

Start the gateway and verify health

Use the built-in daemon controls. Do not skip status checks.

Windows PowerShell

openclaw gateway start
openclaw gateway status
openclaw gateway restart
openclaw gateway status
STEP 4

Run first real terminal sequence

Copy and run this exact sequence to validate end-to-end behavior.

Windows PowerShell

cd $env:USERPROFILE
mkdir openclaw-test -Force
Set-Location openclaw-test
openclaw help
openclaw gateway status
openclaw gateway restart
openclaw gateway status
Set-Location ..
VERIFY

Verification commands

Keep these handy before every major change.

node -v
npm -v
openclaw help
openclaw gateway status
Get-Command openclaw
TROUBLESHOOT

Fast recovery commands

Use these when the command is missing, gateway is stale, or package install is corrupted.

# PATH refresh in current shell
$env:Path = [System.Environment]::GetEnvironmentVariable('Path','Machine') + ';' + [System.Environment]::GetEnvironmentVariable('Path','User')
Get-Command openclaw

# reinstall OpenClaw cleanly
npm uninstall -g openclaw
npm cache verify
npm install -g openclaw

# gateway reset cycle
openclaw gateway stop
openclaw gateway start
openclaw gateway status

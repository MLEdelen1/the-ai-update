n8n setup for reliable AI automations

Stand up n8n, create a production-safe workflow, verify schedule and webhook triggers, and troubleshoot execution errors.

STEP 1

Install n8n and start local server

Windows PowerShell

npm install -g n8n
n8n --version
n8n start

macOS/Linux (optional)

npm install -g n8n
n8n --version
n8n start
STEP 2

Create first workflow with trigger

Use Schedule Trigger for deterministic tests before webhooks.

# no shell command required in UI step
# in browser: open http://localhost:5678
# create workflow: Schedule Trigger -> Set -> Respond to Webhook (or Email node)
STEP 3

Add AI node using local Ollama

Set endpoint to http://localhost:11434 and select your tested model.

curl http://localhost:11434/api/tags
n8n start
STEP 4

Enable persistence and secure auth

$env:N8N_BASIC_AUTH_ACTIVE="true"
$env:N8N_BASIC_AUTH_USER="admin"
$env:N8N_BASIC_AUTH_PASSWORD="change-this-now"
$env:N8N_HOST="localhost"
$env:N8N_PORT="5678"
n8n start
VERIFY

Verification commands

n8n --version
Test-NetConnection -ComputerName localhost -Port 5678
curl http://localhost:11434/api/tags
Get-Process n8n -ErrorAction SilentlyContinue
TROUBLESHOOT

Troubleshooting commands

# port conflict on 5678
Get-NetTCPConnection -LocalPort 5678 | Select-Object -First 5

# clean reinstall
npm uninstall -g n8n
npm cache verify
npm install -g n8n

# test with different port
$env:N8N_PORT="5680"
n8n start

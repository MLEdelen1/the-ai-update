Agent Zero setup for safe autonomous task runs

Set up Agent Zero with controlled permissions, test in a sandbox, verify tool boundaries, and recover quickly if runs drift.

STEP 1

Create isolated project folder

Never test autonomous behavior in a production repository first.

Windows PowerShell

mkdir $env:USERPROFILE\agent-zero-lab -Force
Set-Location $env:USERPROFILE\agent-zero-lab
git init

macOS/Linux (optional)

mkdir -p ~/agent-zero-lab && cd ~/agent-zero-lab
git init
STEP 2

Install runtime + dependencies

Use Python 3.11+ and pip tooling, then install Agent Zero in a dedicated venv.

Windows PowerShell

winget install Python.Python.3.11
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
pip install agent-zero

macOS/Linux (optional)

python3 -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip
pip install agent-zero
STEP 3

Set strict environment variables

Define model endpoint and hard limits before first run.

$env:AGENT_ZERO_MAX_STEPS="10"
$env:AGENT_ZERO_ALLOW_SHELL="false"
$env:AGENT_ZERO_LOG_LEVEL="info"
agent-zero --help
STEP 4

Run safe pilot task

Start with a read-only task. Confirm output quality and boundaries.

agent-zero run "Summarize this folder and suggest one cleanup plan. Do not delete files."
agent-zero run "Draft release notes from git history only."
VERIFY

Verification commands

python --version
pip show agent-zero
agent-zero --help
agent-zero config show
git status
TROUBLESHOOT

Troubleshooting commands

# reset virtual environment
Deactivate
Remove-Item -Recurse -Force .venv
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install agent-zero

# if model calls fail, print environment
Get-ChildItem Env:AGENT_ZERO*

# if permissions are too open, enforce safer defaults
$env:AGENT_ZERO_ALLOW_SHELL="false"
$env:AGENT_ZERO_MAX_STEPS="8"

ORDER

Use this exact integration order

First Ollama, second OpenClaw, third n8n, fourth Agent Zero.

Why: it gives you local model output first, then orchestration, then automation, then advanced agent UI.
STEP 1

Bring up core services

ollama serve
openclaw gateway start
n8n start

Expected result: three services running in separate terminals.

STEP 2

Verify each layer quickly

ollama list
openclaw gateway status
n8n --version
docker ps

Expected result: all commands return healthy output.

STEP 3

Connect n8n to Ollama

In n8n add an HTTP Request node that posts to http://localhost:11434/api/generate.

Expected result: n8n gets local model output.

STEP 4

Use OpenClaw to drive operations

Use OpenClaw commands to manage service lifecycle and run your local routine.

Expected result: one command center for daily operations.

STEP 5

Add Agent Zero for interactive tasks

docker run -p 50001:80 agent0ai/agent-zero

Expected result: Agent Zero UI available at http://localhost:50001 while other stack parts remain active.
